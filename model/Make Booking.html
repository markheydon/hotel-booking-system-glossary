Make Booking

Description

A room in a hotel is booked by a guest.

  1. Guest or Receptionist starts the booking process.
  2. The user selects the hotel, dates of the booking and room types required.
  3. The system shows availability for the dates and room type(s) along with the price.
    1. No availability in this hotel: The system suggests other hotels in the same business with availability.
    2. No availability in any hotel: The system informs the user and terminates the use case.
  4. The user accepts the booking, assuming the price and availability is acceptable.
    1. Guest doesn't like price or what is being offered: Use case terminates.
  5. The system requests the relevant number of guest name and addresses.
  6. The user completes the booking by entering the name and address information of all the guests.
    1. Guest is already known to the business: Continue to next step.
  7. The system creates the booking and gives it a booking number.
  8. The system allocates the rooms as booked.
  9. The user is given the booking number for reference.
